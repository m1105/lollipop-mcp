# 01.5 — Windows 端部署 (DLL + opener)

> Last updated: 2026-04-06

在 Windows 機器上部署 Lollipop 系統：安裝多開器 (opener.py) 和 DLL 注入器。此步驟在 [00 — Windows 環境安裝](00-windows-setup.md) 和 [01 — Lollipop Plugin 安裝](01-install-plugin.md) 之後完成。

---

## 你需要準備

- **已完成 [00 — Windows 環境安裝](00-windows-setup.md)**
  - Python 3.9+ 已安裝，且在 PATH 中
  - npm 可正常執行
  
- **已完成 [01 — Lollipop Plugin 安裝](01-install-plugin.md)**
  - Lollipop 源碼已複製到本地
  - Claude Code MCP 連線驗證通過
  
- **Windows 機器上有 LineageClone 遊戲客戶端**
  - 已下載並安裝遊戲
  - 可以正常啟動遊戲（不須登入，只需驗證啟動）

- **Windows 管理員權限**（啟動遊戲進程時需要）

---

## 安全警告 — 重要！

> **Warning: NCGuard 偵測**
>
> **DLL 絕對不能放在遊戲目錄內。** NCGuard（遊戲內反作弊系統）會在啟動時掃描遊戲目錄的所有 DLL，發現未知 DLL 會立即關閉遊戲並可能封號。
>
> 將 DLL 放在遊戲目錄外的獨立位置（例如 `C:\Lollipop\` 或 `D:\Bot\`）。

> **Warning: Hooks 被偵測**
>
> **不使用任何形式的 hooks、debug registers 或 breakpoints。** NCGuard 會主動檢測：
> - VEH (向量化異常處理)
> - 硬體斷點 (DR0-DR3)
> - 內聯 hooks (inline code patching)
> - IAT hooks
>
> 使用這些方法會導致帳號被鎖定。Lollipop 只使用純記憶體讀取（無代碼修改）和 UI 模擬（點擊按鈕）。

## Part A: 安裝 opener.py (多開器)

opener.py 是一個 Python HTTP 伺服器，負責：
- 管理遊戲帳號設定
- 自動啟動遊戲客戶端
- 注入 DLL 到遊戲進程
- 提供帳號/機器狀態的 API

### Step A1: 複製 stealth 目錄

opener.py 位於 Lollipop 源碼的 `stealth/` 目錄。將整個目錄複製到 Windows 機器。

**從 macOS（如果源碼在 macOS）：**

```bash
# 1. 複製到 Windows 可存取的地方（例如 USB、網路共享、或 OneDrive）
scp -r /Users/m1105mini/Desktop/Bd/Lollipop/stealth/ user@windows-machine:C:\Lollipop\

# 或使用 rsync（推薦，更快）
rsync -av /Users/m1105mini/Desktop/Bd/Lollipop/stealth/ user@windows-machine:C:\Lollipop\
```

**在 Windows 機器上：**

或直接從 GitHub 複製（需要 git 已安裝）：

```powershell
cd C:\
git clone https://github.com/m1105/lollipop.git
cd lollipop
# 現在 C:\lollipop\stealth\ 包含 opener.py
```

驗證目錄結構：

```powershell
dir C:\Lollipop\stealth\
```

預期包含：
```
C:\Lollipop\stealth\
  ├── opener.py                    ← 主程式
  ├── requirements.txt             ← Python 依賴
  ├── setup_python.bat             ← 自動安裝 Python（可選）
  ├── config/
  │   ├── accounts.json            ← 帳號設定（稍後編輯）
  │   ├── servers.json             ← 伺服器列表
  │   └── bot_profiles/            ← bot 配置文件夾
  ├── launcher.exe                 ← DLL 注入器
  └── web/                         ← 網頁 UI（可選）
```

### Step A2: 安裝 Python 和依賴

如果 Windows 上還沒有 Python，有兩種選項：

#### 選項 1: 自動安裝（推薦新手）

```powershell
cd C:\Lollipop\stealth
setup_python.bat
```

此批次檔會自動下載 embeddable Python 並安裝所需的庫。等待完成後，跳到 Step A3。

#### 選項 2: 手動安裝（如果已有 Python）

1. 檢查 Python 版本：

```powershell
python --version
```

確保是 3.9 以上版本。

2. 安裝依賴：

```powershell
cd C:\Lollipop\stealth
pip install -r requirements.txt
```

預期輸出類似：
```
Successfully installed click-8.1.7 cryptography-42.0.2 flask-3.0.0 ...
```

### Step A3: 設定帳號

編輯 `config/accounts.json`，添加你的遊戲帳號。

用任何文字編輯器（例如 VS Code、記事本）打開：

```powershell
notepad C:\Lollipop\stealth\config\accounts.json
```

或在 VS Code 中：

```powershell
code C:\Lollipop\stealth\config\accounts.json
```

編輯內容，格式如下：

```json
[
  {
    "id": "account_1",
    "username": "your_gameaccount@email.com",
    "password": "your_password",
    "server_num": 1,
    "server_page": 1,
    "char_slot": 1,
    "enabled": true
  },
  {
    "id": "account_2",
    "username": "another_account@email.com",
    "password": "another_password",
    "server_num": 2,
    "server_page": 1,
    "char_slot": 1,
    "enabled": true
  }
]
```

**欄位解釋：**

| 欄位 | 說明 | 例子 |
|------|------|------|
| `id` | 帳號唯一 ID（用於 API） | `"bot_1"` 或 `"main"` |
| `username` | 遊戲帳號（郵箱格式） | `"player@example.com"` |
| `password` | 帳號密碼 | `"MySecure123"` |
| `server_num` | 伺服器號碼（1-17） | `1` = 太陽神阿波羅，`2` = 愛神邱比特 等 |
| `server_page` | 伺服器頁數 | `1`（預設） |
| `char_slot` | 角色位置（1-3） | `1` = 第一個角色 |
| `enabled` | 是否啟用 | `true` 或 `false` |

保存檔案（Ctrl+S）。

### Step A4: 啟動 opener.py

在 PowerShell 中，切換到 stealth 目錄並啟動：

```powershell
cd C:\Lollipop\stealth
python opener.py
```

預期輸出：

```
[2026-04-06 10:30:45] Lollipop Opener v1.0
[2026-04-06 10:30:45] Config: C:\Lollipop\stealth\config
[2026-04-06 10:30:45] Listening on http://localhost:8600
[2026-04-06 10:30:45] Web UI: http://localhost:8600/ui
[2026-04-06 10:30:45] Ready to accept connections.
```

如果看到此訊息，opener 已成功啟動。

**保持此 PowerShell 視窗開啟，** 不要關閉（否則 opener 會停止）。

### Step A5: 驗證 opener API

在另一個 PowerShell 視窗中，測試 API 連線：

```powershell
Invoke-WebRequest http://localhost:8600/api/instances | ConvertFrom-Json
```

或使用 `curl`（如果已安裝）：

```powershell
curl http://localhost:8600/api/instances
```

預期回應（JSON 格式）：

```json
{
  "instances": [],
  "status": "ok"
}
```

如果看到此訊息，opener API 正常運作。

## Part B: DLL 部署

DLL 是注入到遊戲進程中的動態連結庫，負責：
- 讀取遊戲內存（位置、怪物、玩家等）
- 執行遊戲操作（移動、攻擊、使用技能等）
- 透過 HTTP API 與 opener/Claude Code 通訊

DLL 部署有兩種方式：自動（從 macOS）或手動。

### 部署方式 A: 自動部署（從 macOS）

如果你在 macOS 上有 Lollipop 源碼，可以使用 `deploy.sh` 自動編譯和部署 DLL。

在 macOS 上，切換到 Lollipop 目錄：

```bash
cd /Users/m1105mini/Desktop/Bd/Lollipop
```

編輯 `deploy.sh`，設定目標 Windows 機器的 IP：

```bash
# deploy.sh 內的設定
WINDOWS_HOST="192.168.1.100"    # 修改為你的 Windows IP
WINDOWS_USER="Administrator"     # Windows 用戶名
WINDOWS_PATH="C:\Lollipop\"      # DLL 部署路徑（不能是遊戲目錄）
```

執行部署：

```bash
bash deploy.sh
```

此腳本會：
1. 上傳 DLL 源碼到 Windows
2. 在 Windows 上遠端編譯 DLL
3. 生成 `dwmapi_ex.dll` 放到 `C:\Lollipop\`
4. 自動更新 opener.py 的 DLL 路徑設定

部署完成後，回到 [Step B3: 驗證](#step-b3-驗證dll注入)。

### 部署方式 B: 手動部署

如果 macOS 上沒有 Lollipop 源碼，或 deploy.sh 無法運作，可以手動部署編譯好的 DLL。

#### Step B1: 準備 DLL

從以下來源取得編譯好的 `dwmapi_ex.dll`：

1. **從現有的 Lollipop 安裝**（如果你已經跑過一次）
   - 位置通常在：`C:\Lollipop\dwmapi_ex.dll`

2. **從 GitHub Release 下載**
   - 前往 https://github.com/m1105/lollipop/releases
   - 下載最新版本的 `dwmapi_ex.dll`

3. **從朋友或社區取得**
   - 詢問有編譯環境的人幫你編譯

#### Step B2: 部署到正確位置

> **重要：選擇部署位置時，務必在遊戲目錄之外。**

**錯誤位置（NCGuard 會掃描並封號）：**
```
C:\Program Files\LineageClone\                    ← 遊戲目錄 ✗
D:\Games\LineageClone\                            ← 遊戲目錄 ✗
```

**正確位置（遊戲目錄外）：**
```
C:\Lollipop\dwmapi_ex.dll                         ← 推薦 ✓
D:\Bot\dwmapi_ex.dll                              ← 可以 ✓
C:\Users\Administrator\AppData\Local\Lollipop\    ← 也可以 ✓
```

將 DLL 複製到選定的位置。例如：

```powershell
# 將 dwmapi_ex.dll 複製到 C:\Lollipop\（建議）
mkdir C:\Lollipop
copy dwmapi_ex.dll C:\Lollipop\dwmapi_ex.dll
```

#### Step B3: 更新 opener 設定

編輯 `config/settings.json`，告知 opener 其中 DLL 路徑：

```powershell
notepad C:\Lollipop\stealth\config\settings.json
```

或在 VS Code 中：

```powershell
code C:\Lollipop\stealth\config\settings.json
```

找到或新增 `bot_dll_path` 設定：

```json
{
  "bot_dll_path": "C:\\Lollipop\\dwmapi_ex.dll",
  "launcher_exe_path": "C:\\Lollipop\\launcher.exe",
  "game_exe_path": "C:\\Program Files\\LineageClone\\GameClient.exe"
}
```

**重要：反斜槓要雙倍（`\\`），或使用正斜槓（`/`）。**

### Step B3: 驗證 DLL 注入

啟動一個帳號，驗證 DLL 是否成功注入。

在 Claude Code（macOS）的終端中，使用 MCP 命令：

```bash
# 啟動第一個帳號
opener_start(acc_id="account_1")

# 等待 30-60 秒（遊戲啟動 + DLL 注入）

# 檢查 DLL 是否連接
bot_list
```

預期輸出：

```
[
  {
    "host": "192.168.1.100",
    "port": 5577,
    "char_name": "你的角色名",
    "email": "account@example.com",
    "hp": 1000,
    "level": 50,
    "class": "Knight",
    "bot": true
  }
]
```

如果看到此列表，表示：
1. 遊戲成功啟動
2. DLL 成功注入
3. DLL HTTP API 在線（port 5577）
4. Lollipop Plugin 可以連線到 DLL

如果沒有出現任何 bot，檢查以下內容：

| 症狀 | 原因 | 解決方案 |
|------|------|--------|
| `opener_start` 執行但 bot_list 仍為空 | 遊戲未啟動或 DLL 未注入 | 檢查 Windows 的 opener 日誌（PowerShell 視窗） |
| 遊戲啟動但立即關閉 | NCGuard 偵測到 DLL 在遊戲目錄 | 移動 DLL 到遊戲目錄外 |
| bot_list 有結果但 port 不對 | launcher.exe 設定錯誤 | 檢查 settings.json 的 launcher_exe_path |
| HTTP timeout（localhost:5577） | 防火牆封鎖 | 允許 Windows 防火牆的 port 5577 進站 |

#### 防火牆設定

如果 DLL HTTP API 超時，檢查 Windows 防火牆是否阻擋了 port 5577。

在 **Windows Defender 防火牆 > 進階設定** 中，新增進站規則：

```
規則名稱: Lollipop DLL API
協議: TCP
本機連接埠: 5577
動作: 允許
```

或在 PowerShell 中（需管理員）：

```powershell
New-NetFirewallRule -DisplayName "Lollipop DLL API" `
  -Direction Inbound -LocalPort 5577 -Protocol TCP -Action Allow
```

## 常見問題

### Q1: 可以在遊戲目錄內部署 DLL 嗎？

**不行。** NCGuard 會掃描遊戲目錄的所有 DLL，發現陌生 DLL 會立即關閉遊戲。必須放在遊戲目錄外。

### Q2: opener 啟動後，但遊戲不會啟動？

檢查以下內容：

1. 確認 `config/accounts.json` 的帳號設定是否正確
2. 檢查 launcher.exe 路徑設定是否正確
3. 查看 PowerShell 視窗的錯誤訊息
4. 手動執行遊戲，確認遊戲客戶端本身能啟動

### Q3: 遊戲啟動了，但 DLL 沒注入？

檢查以下內容：

1. 遊戲視窗標題應顯示 "LineageClone" 且進程存在
2. 檢查 `bot_dll_path` 設定是否指向正確的 DLL 檔案
3. DLL 檔案是否存在且可被 launcher.exe 讀取
4. 防火牆是否封鎖了 port 5577

### Q4: bot_list 為空，但遊戲在執行？

等待更久一點（30-60 秒），DLL 掃描需要時間。如果仍為空：

1. 檢查 `curl http://localhost:5577/stats`，DLL HTTP API 是否在線
2. 如果 curl timeout，檢查防火牆設定
3. 檢查 opener 日誌，是否有 DLL 注入錯誤

### Q5: 可以同時啟動多個帳號嗎？

可以。設定多個帳號在 `config/accounts.json`，然後：

```bash
opener_start(acc_id="account_1")
opener_start(acc_id="account_2")
opener_start(acc_id="account_3")
```

每個帳號會啟動獨立的遊戲進程，DLL 會分配不同的 port（5577, 5578, 5579...）。

## 技術參考

### DLL HTTP API 端點

DLL 在 `http://localhost:PORT/` 提供以下端點：

| 端點 | 說明 | 例子 |
|------|------|------|
| `/stats` | bot 狀態（HP、MP、位置等） | `GET /stats` |
| `/entities` | 周圍怪物/玩家 | `GET /entities` |
| `/inventory` | 背包物品 | `GET /inventory` |
| `/position` | 當前座標 | `GET /position` |
| `/cmd` | 執行命令 | `POST /cmd {"action":"move","x":100,"y":200}` |

詳見 [Lollipop MCP 參考](../reference/api.md)。

### opener.py API 端點

opener 在 `http://localhost:8600/` 提供以下端點：

| 端點 | 說明 | 例子 |
|------|------|------|
| `/api/instances` | 列出所有遊戲進程 | `GET /api/instances` |
| `/api/start` | 啟動帳號 | `POST /api/start {"acc_id":"account_1"}` |
| `/api/stop` | 停止帳號 | `POST /api/stop {"acc_id":"account_1"}` |
| `/api/status` | 帳號狀態 | `GET /api/status?acc_id=account_1` |

詳見 [opener API 參考](../reference/opener-api.md)。

### 環境變數

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `LOLLIPOP_OPENER_PORT` | opener 監聽的 port | `8600` |
| `LOLLIPOP_DLL_PORT_START` | DLL 起始 port | `5577` |
| `GAME_EXE_PATH` | 遊戲客戶端完整路徑 | (自動偵測) |
