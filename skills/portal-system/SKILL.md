## 用途

查詢已知地圖傳送門的位置與目的地。傳送門是地圖上的特殊地板格（flags=2 或 flags=11），
用於在地圖之間移動。client 端只儲存傳送門格位置，實際目的地由 server 決定。

## 常見操作

### 查詢所有已知傳送門

```
bot_portal_db()
```

回傳所有已記錄的傳送門，包含：
- `map_id`: 所在地圖 ID
- `x`, `y`: 世界座標
- `dest_map`: 目的地地圖名稱（若已知）
- `dest_x`, `dest_y`: 目的地座標（若已知）

### 查詢特定地圖的傳送門

```
bot_portal_db(map_id=0)
```

常用 map_id：
- 0 = 說話之島
- 1 = 說話之島地監
- 4 = 大陸
- 7 = 古魯丁地監
- 53 = 奇岩地監

### 查詢到特定目的地的傳送門

```
bot_portal_db(dest="古魯丁")
```

## 注意事項

- 傳送門資料庫是靜態資料，不反映即時狀態
- 部分傳送門目的地未知（server-side only）
- 實際走進傳送門使用 bot_smartmove 或 bot_walk 移動到傳送門座標
- 地監入口座標參考（常用）：
  - 說話之島地監: (32478, 32851)
  - 古魯丁地監: (32728, 32929)
  - 奇岩地監: (33311, 33061)
  - 龍之谷入口: (33239, 32453)
